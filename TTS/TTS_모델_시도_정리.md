# TTS 모델 시도 이력 및 문제 정리

## 목적
루게릭병(ALS) 환자 음성 복원 — 과거 음성 파일(참조 음성)을 업로드하면, 텍스트를 그 목소리로 읽어주는 웹 서비스 구축

---

## 시도한 모델 및 문제 이력

### 1. XTTS v2 (coqui-tts) — 1차 시도
**패키지:** `TTS==0.22.0`

| 항목 | 내용 |
|------|------|
| 결과 | ❌ 설치 실패 |
| 원인 | PyPI에 해당 버전 없음 (`No matching distribution found`) |
| 조치 | 버전 핀 제거 후 `TTS` 재시도 → 동일 오류 (PyPI에서 패키지 자체가 내려감) |

---

### 2. XTTS v2 (coqui-tts) — 2차 시도
**패키지:** `coqui-tts` (최신 버전 0.27.5 자동 설치)

| 항목 | 내용 |
|------|------|
| 결과 | ❌ 실행 오류 |
| 원인 1 | `transformers 5.2.0`에서 `isin_mps_friendly` 함수 삭제 → `cannot import name 'isin_mps_friendly'` |
| 원인 2 | `coqui-tts 0.27.5`는 `transformers>=4.57` 요구하지만, 4.57.x에는 `is_torch_greater_or_equal` 함수 없음 |
| 결론 | coqui-tts 0.27.5는 어떤 transformers 버전과도 충돌 — **패키지 자체 버그** |

---

### 3. F5-TTS — 대안 시도
**패키지:** `f5-tts`, `openai-whisper`

| 항목 | 내용 |
|------|------|
| 결과 | ⚠️ 설치는 성공, 한국어 품질 불량 |
| 원인 | F5-TTS는 주로 **영어** 데이터로 학습됨. 한국어 텍스트를 입력해도 전혀 다른 내용을 발화함 |
| 조치 | 한국어 지원이 명확한 XTTS v2로 복귀 결정 |

---

### 4. XTTS v2 (coqui-tts) — 3차 시도 ✅ 현재 사용 중
**패키지:** `coqui-tts==0.26.2`

| 항목 | 내용 |
|------|------|
| 설치 | ✅ 성공 |
| transformers | 4.51.3 (충돌 없음) |

#### 겪은 문제들 (해결 완료)

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| 라이센스 동의 프롬프트 blocking | coqui-tts 최초 실행 시 y/n 입력 요구 | `COQUI_TOS_AGREED=1` 환경변수 설정 |
| M4A 파일 포맷 오류 | XTTS가 M4A 직접 읽기 불가 | `pydub` + `ffmpeg`로 WAV 자동 변환 |
| 한국어 오류 | `hangul_romanize` 패키지 없음 | `pip install hangul_romanize` |
| JSON 파싱 오류 (프론트) | 변환 오류가 try/except 밖에서 발생 → plain text 500 반환 | to_wav() 호출을 try/except 안으로 이동 |

#### 현재 남은 문제

| 문제 | 증상 | 원인 분석 |
|------|------|-----------|
| **텍스트 후반부 할루시네이션** | "안녕하세요, 김예린입니다" 입력 시 "안녕하세요"만 정확히 발화하고 이후는 엉뚱한 말 생성 | XTTS v2의 autoregressive 생성 특성상 **참조 음성 길이 대비 생성 길이가 길어지면 안정성 저하** 발생 |

## 남은 과제 및 개선 방향

### 할루시네이션 문제 해결 방향

| 방법 | 설명 |
|------|------|
| **참조 음성 길이 조정** | XTTS v2는 6~12초 참조 음성이 오히려 최적. 30초는 너무 길 수 있음 |
| **긴 텍스트 분할 생성** | 문장 단위로 쪼개서 각각 생성 후 이어붙이기 |
| **temperature 파라미터 낮추기** | `tts_to_file()` 호출 시 `temperature=0.7` 등 낮춰서 안정성 확보 |
